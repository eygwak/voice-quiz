# 🎯 음성 스피드 퀴즈 앱 – 기능 요구사항서 (v0.3)

## 1. 앱 목적

- 사용자는 **말로 정답을 맞히는 스피드 퀴즈**를 즐긴다  
- AI는 출제자 또는 추측자로 행동한다  
- 짧은 플레이(1~3분)로 반복 플레이가 가능해야 한다  
- **앱 내 모든 UI 텍스트, 음성 안내, 퀴즈 콘텐츠는 영어로 제공된다**

---

## 2. 핵심 게임 모드 (MVP)

> v0.3부터 **두 모드 모두 MVP에 포함**한다.

### 2.1 AI가 설명 → 사용자가 맞히기 (메인 모드)

**(MVP 필수)**

#### 기본 흐름

1. 라운드 시작  
2. AI가 단어를 설명한다 (**영어 음성**)  
3. 사용자가 끼어들어 정답을 말한다  
4. 정답이면 즉시 다음 단어로 이동  
5. 오답 또는 Close인 경우, **AI는 설명을 이어서 계속 제공**한다  
6. 제한 시간 종료 시 결과 화면 표시  

#### 규칙

- 라운드 시간: 60초  
- 패스 가능 횟수: 2회  
- 정답 개수 = 점수  

#### 추측자 규칙

- **정답을 맞히는 사람(사용자)은 질문할 수 없고, 오직 정답을 말하는 방식으로만 추측한다**
  - 예: “Is it ...?” 같은 질문 금지
  - 오직 정답 후보를 말하기만 허용

#### AI 설명 규칙

- 단어 자체, 철자, 직접적인 동의어 사용 금지  
- “You use this when…”, “You usually see this in…”과 같은  
  **간접적이고 자연스러운 영어 설명만 허용**  
- 설명은 한 번에 길지 않게, 필요 시 여러 번 나누어 제공  

---

### 2.2 사용자가 설명 → AI가 맞히기 (역방향 모드)

**(MVP 필수)**

#### 기본 흐름

1. 사용자가 단어를 확인 (**영어**)  
2. 사용자가 **자유롭게 영어 음성으로 설명**한다  
3. AI가 영어로 추측 발언  
4. 사용자가 버튼을 통해 판정한다  
   - Correct  
   - Incorrect  
   - Close  
   - **Close를 선택해도 AI는 질문하지 않으며, 계속 추측만 한다**
5. 제한 시간 내 맞히면 성공  

#### 규칙

- 라운드 시간: 60초  
- 단어(문제) 수: 라운드 내 가능한 만큼  
- 패스 가능 횟수: 2회  
- 성공(정답)한 단어 수 = 점수  

#### 추측자(AI) 규칙

- **정답을 맞히는 AI는 질문을 할 수 없고, 오직 정답 후보를 말하는 방식으로만 추측한다**
  - 예: “Is it a fruit?” 같은 질문 금지
  - 오직 “I think it is …” 형태의 추측만 허용

#### 사용자 입력 규칙 (출제자 역할)

- 사용자는 **자연스럽게 연속 발화**하며 설명할 수 있다  
- AI의 추측은 사용자의 발화를 기준으로 적절한 간격으로 이루어진다  
- 사용자가 “Correct”로 판정하면 즉시 다음 단어로 넘어간다  

---

## 3. 사용자 입력 방식

> **MVP 테스트 단계에서는 STT/TTS 모두 iOS 내장 무료 기능을 사용한다.**
> - STT: Apple Speech Framework (온디바이스)
> - TTS: AVSpeechSynthesizer


- **음성 입력이 기본**
- 음성 인식 결과는 **실시간 자막**으로 화면에 표시
- 말하는 중에도 언제든 정답 판정 가능
- 선택적 보조 입력:
  - 패스 버튼
  - 키보드 입력 허용(옵션)

---

## 4. 정답 판정 UX

### 정답 상태

- **정답 (Correct)**  
- **오답 (Incorrect)**  
- **유사함 / 근접 (Close)**  

### 판정 기준 (사용자 관점)

- 발음이 다소 부정확해도 정답으로 인정될 수 있다  
- 일상적으로 사용되는 유사 표현도 정답으로 인정될 수 있다  

### 피드백 방식

- 음성 피드백 (영어)
  - “Correct!”
  - “Nice try!”
  - “Close!”
- 시각적 피드백
  - 색상 변화
  - 짧은 애니메이션
- 효과음 포함

---

## 5. 단어 및 콘텐츠 구조

### 단어 속성

- 단어 (영어)  
- 카테고리  
- 난이도(테스트 단계에서는 출제 로직에 미반영)  
- 금칙어 목록 (직접적인 힌트 방지용)  
- 동의어 목록(synonyms, 판정 보정용)
